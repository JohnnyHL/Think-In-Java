代理模式
什么是代理模式？就是供应商jobber很忙，jobber没空，要找jobber谈事，请先跟jobber的代理人谈。
那么代理人总得知道jobber能做哪些事情，哪些事情不能做，此时供应商jobber和代理人Proxy之间就会约定一个共同的接口。
代理人proxy只是传话作用，不用干实际的事情，而是由jobber去搞具体的事情。

1.jobber和Proxy之间的接口Concersation类，约定了供应商要做哪些事情。
2.供应商jobber(被代理人)，实现接口。
3.代理人Proxy，可以代理N多个供应商jobber，只是根据用户需要见哪个，就安排哪个。
4.用户User，将与代理人发起此次会谈，实际的事情却是由jobber来做，代理人仅仅是个空壳传话筒。

总结：
代理模式主要使用了多态，干活的是被代理类，代理类就是接单子接业务；
用户让代理人干活，OK，代理人就安排幕后的供应商jobber去做具体的事情；
那怎么晓得被代理类jobber能不能搞呢，知根知底啊，代理人和被代理人之间约定的共用一个接口，你能干啥，他能做啥都一清二楚。
